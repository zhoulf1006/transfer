# ADR-0001: 局域网直连,无服务器、无账号

- 状态: 已接受

## 背景与问题

Transfer 是跨平台(macOS/Windows)局域网工具:文件传送 + 截屏标注 + 笔记。需要决定基础连接架构:设备之间如何找到彼此并传输数据。

## 备选项

> 注:本决策早于文档体系,备选对比未留档;以下第 2 项是定位上被排除的方向,非当年的对比记录。

1. **局域网直连**(无服务器、无账号,协议对齐 LocalSend)
2. 云中转/账号体系——与产品定位冲突:引入服务器成本、运维负担与隐私暴露面,且局域网场景下纯属绕路

## 决策

选定**局域网直连**:Electron + TypeScript,多播/广播发现 + HTTP(S) 直传,零服务器、零账号。

## 后果

- 正面:零云成本、零账号,隐私天然可控。
- 负面:发现依赖局域网多播/广播,企业网/代理环境下脆弱(ADR-0005/0006 逐步补强);信任无 CA 可依,走自签证书 + TOFU(ADR-0004)。
