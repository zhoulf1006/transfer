# ADR-0008: 截图功能范围——三出口,对齐 Snipaste 标注,不做通用截图软件

- 状态: 已接受

## 背景与问题

截图定位为「截图 → 直接发给对方」的轻量工具,与传输/聊天场景强绑定。需要划清与 Snipaste 类通用桌面截图软件的功能边界,防止范围蔓延。

## 备选项

1. **轻量强绑定**:三出口(发送到聊天/剪贴板/文件)+ 标注工具对齐 Snipaste(全套标注),其余不做
2. 对标 Snipaste 全集(通用截图软件)——否决:偏离"发给对方"主场景,功能包袱重
3. 逐项范围裁决(调研 Snipaste 官方 wiki 后与用户确认):**明确不做**——钉图、OCR、滚动/长截图、录屏/GIF、延时截图;马克笔/荧光笔为用户点名不做

## 决策

选定**方案 1**:三出口 + 全套标注;方案 3 的不做清单为硬边界。

## 后果

- 正面:功能面收敛,截图始终服务于"发给对方"的主场景。
- 中性:用户要通用截图能力时引导用专业工具;P2 项(如滚动长图)进入路线图暂缓区,重启需新决策。
