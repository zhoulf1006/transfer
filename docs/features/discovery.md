# 设备发现与连接

> 关联决策: [ADR-0006](../adr/0006-broadcast-fallback-dual-channel.md)、[ADR-0004](../adr/0004-https-self-signed-tofu.md)

## 概述

局域网内两台设备要互传,先得找到彼此——手动输 IP 门槛高且 IP 常变。打开 Transfer 即自动发现:同网设备数秒内出现在列表里,无需任何配置。

## 能力

- 同一局域网内自动发现,免配置;兼容官方 LocalSend 客户端的发现广播
- 设备列表实时反映在线状态;失联设备灰显置底保留一段时间,超期自动移除
- 离线设备的保留时长可在设置中调整(0 = 永不移除)
- 设备身份跨重启稳定:对方改名、换 IP 后仍识别为同一设备(备注、聊天历史不丢)
- 设备间传输全程加密,可防局域网内被动窃听

## 边界与不做

- 需双方在同一局域网且都运行 Transfer;不做跨网发现、不做服务器中转
- 任一方开启 VPN 全隧道会阻断直连:能看到对方但发送失败(约 10 秒后提示连接超时);需关 VPN 或开启 Split Tunnel/允许本地子网
- 防冒充依赖接收前的人工确认(设备身份的广播本身未加密,同网段主动攻击者可冒充对端,与官方 LocalSend 同级局限);建议仅在可信局域网使用
